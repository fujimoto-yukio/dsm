# DSM Workbench
## はじめに
DSM Workbenchと名付けて、DSM(Design Structure Matrix)の編集・分析ツールを公開することにしました。
まず最初に、このツールの生い立ちから、開発、公開に至った経緯を簡単に書きます。

開発者である私は1961年生まれ（これを書いている現在63歳）で、元々は機械エンジニアでしたが、SI業界に転職して以来、主に設計開発領域のSIを担当させていただきました。
60歳定年を2年後に控えた2019年1月、（それまでお客さまだった）自動車メーカーの情報システムに出向し、社内SEとして自動車の設計支援システムを中心に、（楽しく）仕事をさせていただきました。
その活動の中のひとつに、DSMの設計活用というテーマもありました。

出向から1年と少し経過した2020年にコロナ禍が発生し、その年のゴールデンウィークは、緊急事態宣言で自宅引きこもりとなりました。あまりに暇でやることがない連休、思いついたのが「DSMを画面で操作できるアプリを作ってみよう！」でした。

SI業界に転職した当初はJavaのプログラミングもやっていましたが、月日が経つにつれて管理の仕事が中心となり、ちょこっとしたツールぐらいしか作らなくなっていたところに、随分と進化したJava（ラムダ式は、サンプルコードを見てもまるで意味不明でした）と（GUIアプリなんか作ったこともないのに）初めてのJavaFXを使って、DSM Workbenchの前身になるDSMアプリのプロトタイプを作りました。
作ったアプリは、CSVからDSMを取り込み、要素を上下（左右）に移動、並び替えるだけの簡単なものでしたが、画面上でDSMを直接操作し整理することができる、画期的なアプリに思えました。

その後、出向から帰任する2022年9月末までの間、仕事の暇を見つけては、画面の見た目や操作性を上げ、可到達マトリクスやグラフデータベースと連携したコミュニティ分析等、思いつくままに機能を追加してみました。
設計現場でも使ってもらいましたが、ベースが1週間で作ったプロトタイプでもあり、業務に本格適用するものにはなりませんでした。

出向から自社に復帰してから1年後の2023年9月末、63歳半で退職することにしました。
ゴールデンウィークよりもはるかに長い退職後の時間を何に使うか考えた時、やりっぱなしになっているDSMツールをきちんと作り直すことに決めました。
JDK1.4か1.5で止まっていたJavaの知識に、Java21までの進化を学び直し、JavaFXによるGUIアプリ開発の基礎も学んで、まるっきり新しいアプリケーションとしてDSM Workbenchを開発しました。

どうしても基盤のコーディング比率が大きくなり楽しみが少なくなりがちな中、個人的な興味で機能を作り込んだところも多々あります。
特に、グラフデータベースやグラフ描画の機能に関しては、グラフ理論の理解はさておき、「何か使えそう」というか、「出来上がって動くものを見るのが楽しい」ということを優先して機能実装しました。

このようにして、実業務の要求とは別に出来上がったDSM Workbenchには、意味のある機能と意味のない機能（使える機能と使えない機能）の玉石混合になっていると思いますので、それを理解した上で使ってください。
作者としては、玉が少しでも混ざっていて、少しでもお役に立てることを期待していますが、どうでしょう？
また、ひとりで設計、開発、テストを行ってきましたので、バグ、不具合もまだまだ残っているはずです。ご連絡いただければ、可能な限り修正しますので、それも覚悟の上使っていただければと思います。
あくまで「As Is」ですが、これらをご理解の上で利用していただければ幸いです。
## 機能説明
DSM Workbenchは、二値型（Binary）と数値型（Numeric）の2種類のDSMを扱うことができます。主要な機能について、以下に記載します。
### DSM Workbench
  - アプリケーションを起動すると、最初に表示される画面です。
  - DSMの種類を指定して、新規のDSMを編集するための`DSM Editor`を起動することができます。
  - ファイルを指定して`DSM Editor`を起動することができます。
  - ファイルを指定して`Graph Visualizer`を起動することができます。
    <img width="712" alt="スクリーンショット 2024-10-09 20 25 13" src="https://github.com/user-attachments/assets/98b0f952-4345-4b41-b90d-32d55dc5440b">
### DSM Editor
  - DSMの編集と分析を行うための画面です。
  - `二値型（Binary）`と`数値型（Numeric）`の2つのDSM形式を取り扱います。
  - 操作モードには、`セル編集モード`と`マトリックス編集・解析モード`の2つのモードがあります。
  - `セル編集モード`では、DSM要素の定義、属性編集とDSM要素間のフィード値を編集することができます。
  - `マトリックス編集・解析モード`では、DSM行列に対する移動、並び替え、部分抽出等の操作と、DSMの順序付けやコミュニティ分析等の解析を行います。
    <img width="1112" alt="スクリーンショット 2024-10-09 20 24 27" src="https://github.com/user-attachments/assets/ce3c534f-70be-48a4-ba24-576a4ed5b702">    
### Graph Visualizer
  - `DSM Editor`で作成したDSMをネットワーク・グラフに描画します。
  - `JUNC Visualizer`と`SmartGraph Visualizer`の2種類のグラフ描画形式を選択することができます。
    <img width="922" alt="スクリーンショット 2024-10-09 20 27 21" src="https://github.com/user-attachments/assets/c5a50f20-0f67-4db4-a477-36470e7e1035">
### Preference Dialog
  - `DSM Workbench`の各種設定値を管理します。
  - 設定値はユーザ毎に保存されます。
    <img width="1117" alt="スクリーンショット 2024-10-09 20 27 36" src="https://github.com/user-attachments/assets/0140a362-64e3-4654-907c-ec7cfe9ddcb1">
### Information DIalog
  - `DSM Workbench`で取り扱う各種情報を詳細表示します。
  - 画面の表示内容をファイルに保存することもできます。
